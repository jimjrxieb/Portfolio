# Phase 6: Shift-Left Cascade (Continuous)

This is what makes jsa-monitor more than a watcher. **Every runtime finding
generates prevention that flows back upstream.** The goal: each issue is
detected at runtime ONCE, then prevented at earlier stages forever.

## The Shift-Left Loop

```
RUNTIME FINDING (jsa-monitor detects at T+0)
  │
  ├─► Response (auto-fix or escalate)
  │
  ├─► Cascade to jsa-devsec
  │   "This should have been caught at code review / CI."
  │   Artifacts: Conftest rule, CI gate, Helm value fix, pre-commit hook
  │   Destination: /tmp/jsa-cascade/devsec/
  │
  ├─► Cascade to jsa-infrasec
  │   "This should have been blocked at admission / cluster hardening."
  │   Artifacts: Kyverno policy, NetworkPolicy, PSS label, admission exception
  │   Destination: /tmp/jsa-cascade/infrasec/
  │
  └─► Cascade to jsa-monitor (self-tuning)
      "This detection can be made faster / more accurate."
      Artifacts: Falco rule refinement, allowlist update, alert threshold
```

## Cascade Mapping

Every auto-fixable pattern maps to specific upstream prevention:

### Crash Fixes → Prevention

| Runtime Fix | jsa-devsec Prevention | jsa-infrasec Prevention |
|------------|----------------------|------------------------|
| OOM fix (memory ×1.5) | Set correct memory limits in manifest/Helm values | LimitRange default review — may be too low |
| ReadOnlyFS emptyDir | Add emptyDir mounts to Helm chart | Golden path template includes common emptyDirs |
| ImagePull rollback | CI must verify image exists + pullable before deploy | imagePullSecrets in namespace template |
| ConfigMap missing | CI validate: all referenced ConfigMaps exist in manifests | Admission: warn on missing config references |
| Health probe fix | Correct probe path/port in manifest | Admission: require probes on all deployments |
| SecurityContext patch | Manifest must include securityContext | Admission: enforce securityContext fields |

### Security Threats → Prevention

| Runtime Detection | jsa-devsec Prevention | jsa-infrasec Prevention |
|------------------|----------------------|------------------------|
| Shell in container | Use distroless base image (no /bin/sh) | Falco allowlist tuned for legitimate shells |
| Sensitive file read | Don't mount sensitive paths | readOnlyRootFilesystem enforced |
| Outbound C2 connection | No curl/wget in production image | Egress NetworkPolicy (deny by default) |
| Crypto mining | Image from trusted registry only | Image signing + verification at admission |
| Privilege escalation | Drop ALL capabilities in manifest | disallow-privilege-escalation policy |
| Persistence | readOnlyRootFilesystem in manifest | AppArmor/seccomp profiles enforced |
| Secret exposure | Use ESO, not K8s secrets in code | automountServiceAccountToken=false |
| Container escape | No hostPath, hostPID, hostNetwork | disallow-host-namespaces policy |
| Data exfiltration | No data transfer tools in image | Egress NetworkPolicy, rate limiting |
| Supply chain | Cosign signing in CI + SBOM | Signature verification at admission |

### Drift → Prevention

| Runtime Detection | jsa-devsec Prevention | jsa-infrasec Prevention |
|------------------|----------------------|------------------------|
| securityContext removed | Conftest rule: deny if no securityContext | Kyverno: require securityContext |
| NetworkPolicy deleted | CI gate: deny if no NetworkPolicy in manifest | Kyverno: require NetworkPolicy per namespace |
| PSS label changed | CI gate: deny PSS downgrade | Kyverno: mutate PSS to minimum baseline |
| Resource limits removed | Conftest: deny if no limits | LimitRange as backstop |

## Cascade Failure Detection

The most important shift-left signal: **when the same issue appears in BOTH
pre-deploy and runtime, the prevention pipeline FAILED.**

```python
# Cross-reference FindingsStore
cascade_failures = findings_store.get_cross_agent_patterns()

# For each:
# - jsa-devsec detected it at commit/CI time
# - jsa-monitor detected it at runtime
# - Means: CI gate didn't block it OR admission didn't enforce it

# Force to B-rank: human must investigate WHY prevention failed
for failure in cascade_failures:
    findings_store.update_rank(failure.id, "B")
    escalate_to_human(
        summary="Cascade failure: prevention pipeline gap",
        details={
            "finding": failure.title,
            "devsec_detection": failure.devsec_date,
            "runtime_detection": failure.runtime_date,
            "gap": "CI gate or admission control didn't block this"
        }
    )
```

## Artifact Generation

### Conftest Rules (→ jsa-devsec)

```rego
# Auto-generated by jsa-monitor cascade
# Source: OOMKilled on deployment/api-server in production
# Date: ${DATE}

deny[msg] {
  input.kind == "Deployment"
  container := input.spec.template.spec.containers[_]
  not container.resources.limits.memory
  msg := sprintf("Container %s must have memory limits set (cascade from runtime OOMKilled)", [container.name])
}
```

### Kyverno Policies (→ jsa-infrasec)

```yaml
# Auto-generated by jsa-monitor cascade
# Source: ReadOnlyFS crash on nginx pod
# Date: ${DATE}
apiVersion: kyverno.io/v1
kind: ClusterPolicy
metadata:
  name: cascade-require-emptydir-for-nginx
spec:
  validationFailureAction: Audit  # start in audit, promote later
  rules:
    - name: require-cache-emptydir
      match:
        resources:
          kinds: [Pod]
      preconditions:
        all:
          - key: "{{ request.object.spec.containers[].image }}"
            operator: AnyIn
            value: ["nginx*"]
      validate:
        message: "Nginx pods with readOnlyRootFilesystem need emptyDir for /var/cache/nginx"
        pattern:
          spec:
            volumes:
              - name: cache
                emptyDir: {}
```

## The End State

Over time, as the cascade loop operates:

```
Month 1: jsa-monitor catches 100 runtime issues, generates 100 cascade artifacts
Month 2: jsa-devsec/jsa-infrasec apply artifacts, block 60 at pre-deploy
Month 3: Only 40 reach runtime. New cascade artifacts generated.
Month 4: Only 20 reach runtime.
Month 6: Only 5 reach runtime (truly novel patterns).

Prevention rate: 5% → 40% → 60% → 80% → 95%
```

The system gets SMARTER over time without retraining any model.
The intelligence is in the cascade artifacts, not the AI.
