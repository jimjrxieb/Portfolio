# CBBP: Comply, Build, Break, Prove

This folder documents the security engineering lifecycle for Portfolio-Prod.

| Phase | Purpose | Status |
|---|---|---|
| [COMPLY](COMPLY/) | Scope the system, map controls, identify gaps, and route work. | Started |
| [BUILD](BUILD/) | Implement preventive controls and remediation work. | Started |
| [BREAK](BREAK/) | Validate whether controls hold under runtime and adversarial testing. | Started |
| PROVE | Package final evidence, summaries, and POA&M artifacts. | Folder reserved |

The operating principle:

```text
A control can be documented and still be ineffective until it has been validated.
```

COMPLY creates the map. BUILD changes the system. BREAK tests the claims. PROVE
packages the evidence or records the risk decision.
