# Career Background — Who Sheyla's Owner Is

## The Origin Story

The person behind this portfolio has 13 years of hands-on compliance experience in OSHA and HAZWOPER (Hazardous Waste Operations and Emergency Response). That background is not a detour into IT security — it is the direct foundation of it. Thirteen years of asking "why is this considered best practice?" and "how do I prove it to an auditor?" is exactly the mindset that makes a good security engineer.

The IT security journey started in a flight simulator server room. Working around that infrastructure sparked the realization that the same discipline used to document OSHA compliance programs — evidence-driven, traceable, never just "trust me" — applied directly to securing systems. If you can document a safety procedure well enough for a federal auditor, you can document a security control well enough for a 3PAO.

## Claude Code and Codex

Claude Code (Anthropic's CLI) has been the primary development tool for almost two years, used nightly. That is not a casual user — that is someone who has built real systems, debugged production failures, and learned how to collaborate with AI as a genuine engineering partner rather than a novelty.

Codex (OpenAI's coding assistant, primarily via GitHub Copilot CLI mode) complements Claude Code for specific code generation tasks. The workflow is: Claude Code for architecture decisions, complex reasoning, and multi-file changes; Codex for rapid code generation and automation scripts.

The key insight from nearly two years of this work: AI tools do not replace judgment. They accelerate execution. The senior engineer still decides what to build and why. The AI executes faster than typing and catches more edge cases than any single human reviewer.

## The Pivot

The career pivot is IT Security + AI Engineering, not a departure from compliance — a deepening of it.

- **Compliance DNA** → becomes the GRC pillar (BERU model, NIST 800-53, FedRAMP evidence packaging)
- **"Why is this best practice?"** → becomes the Break pillar (CySA+ certification path, red-teaming, adversarial thinking)
- **"How do I prove it?"** → becomes the Prove pillar (auditor-ready artifacts, POA&M, CISO reports)
- **Flight simulator server room** → becomes the Build pillar (DevSecOps, Kubernetes, CI/CD hardening)

The CySA+ certification is the formal credential for the Break pillar. The DevSecOps workflow — building, scanning, deploying securely — is the Build pillar in practice every single day.

## Current Focus

Two parallel tracks:

1. **GRC Engineering** (GP-CONSULTING): encoding 13 years of compliance into NIST 800-53 control playbooks, evidence APIs, and auditor-ready output. Not theoretical — actual controls, actual evidence, actual gap analysis.

2. **AI Engineering** (GP-MODEL-OPS): three fine-tuned models (JADE, KATIE, BERU), a RAG pipeline with 33k+ documents, and an ML classifier. All built from scratch using open source tooling. No pre-built AI products — actual model training, evaluation, and deployment.
